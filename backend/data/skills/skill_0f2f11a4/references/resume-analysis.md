# 简历分析指南

> 本指南说明如何从简历中提取关键信息，为面试题生成提供素材。

---

## 分析维度

### 1. 基础信息提取

| 提取项 | 说明 | 用途 |
|-------|------|------|
| 工作年限 | 第一份工作到最近一份工作的时间跨度 | 判断经验深度 |
| 行业背景 | 每份工作的行业分类 | 匹配行业维度 |
| 岗位变迁 | 历任岗位的级别变化（专员→主管→经理） | 判断成长性 |
| 公司规模 | 大厂/中型/小型/创业公司 | 判断候选人背景 |

### 2. 关键事件挖掘

从简历中提取可用于追问的素材：

- **业绩数字**：如"销售额提升30%"、"管理10人团队" — 追问具体做法和背景
- **项目经验**：描述的项目 — 追问个人贡献、遇到的困难、最终结果
- **技能标签**：列出的专业技能 — 追问实际应用深度
- **离职原因**：如能推断 — 设计考察稳定性的问题

### 3. 风险点识别

| 风险类型 | 识别方法 | 对应考察维度 |
|---------|---------|-------------|
| 频繁跳槽 | 每份工作不足1年 | 基本面（稳定性） |
| 空档期 | 工作之间有明显间隔 | 基本面（资历条件） |
| 岗位停滞 | 多年同一级别无晋升 | 底层逻辑（冠军基因） |
| 跨行业 | 最近一次转行 | 愿力（行业认同） |
| 经历模糊 | 简历描述笼统无细节 | 行动力（执行力） |
| 学历相关 | 与岗位要求有差距 | 基本面 |

### 4. 亮点挖掘

从简历中找可用于正向追问的素材：

- **突出业绩**：可深挖成功经验
- **快速晋升**：可考察成长驱动力
- **跨界经历**：可考察学习力和适应力
- **项目主导**：可考察领导力和决策力

---

## 简历→维度匹配逻辑

```
简历特征                    →  优先考察维度
─────────────────────────────────────────
有行业相关经验 ≥ 1年        →  愿力（行业认同）⭐
有跨行业/转行经历            →  愿力（行业认同）⭐⭐
频繁跳槽（<1年/份）          →  基本面（稳定性）⭐⭐
有空档期                    →  基本面（资历条件）⭐⭐
简历描述笼统                →  行动力（执行力）⭐⭐
有管理经验                  →  底层逻辑（将军潜质）⭐
有业绩数据                  →  底层逻辑（冠军基因）⭐
无相关经验                  →  学习力 + 愿力 ⭐⭐
有详细项目经验              →  岗位职责相关 ⭐⭐
```

---

## 输出要求

分析完简历后，输出一个结构化摘要供后续出题使用：

```
## 简历分析摘要

### 候选人画像
- 工作年限：X年
- 行业背景：XXX → XXX → XXX
- 岗位变迁：XXX → XXX → XXX
- 当前状态：在职/离职

### 关键事件
1. [事件1：具体描述]
2. [事件2：具体描述]

### 风险点
- [风险1：描述 + 建议考察维度]
- [风险2：描述 + 建议考察维度]

### 亮点
- [亮点1：描述 + 建议深挖方向]

### 建议考察维度（按优先级）
1. [维度名] — 原因
2. [维度名] — 原因
```
