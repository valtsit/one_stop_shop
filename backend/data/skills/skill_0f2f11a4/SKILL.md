# 面试题目设计 Skill

根据候选人的岗位和简历，生成针对性的结构化面试题。

## 工作流程

按以下四步执行，每步输出中间结果供用户确认。

### Step 1: 简历分析

读取简历内容（文字粘贴或文件），执行以下分析：

1. **提取基础信息**：工作年限、行业背景、岗位变迁、当前状态
2. **挖掘关键事件**：业绩数字、项目经验、技能标签
3. **识别风险点**：频繁跳槽、空档期、岗位停滞、经历模糊
4. **识别亮点**：突出业绩、快速晋升、跨界经历

如果用户提供的是文件（PDF/Word），先读取文件内容再分析。

输出简历分析摘要（格式见 `references/resume-analysis.md`）。

### Step 2: 维度匹配

根据岗位类型和简历特征，从维度库中选择适用的考察维度。

**维度库位置：** `references/dimensions.md`

**维度选择规则：**
- 必选维度：根据岗位类型从维度选择指南中确定
- 加选维度：根据简历风险点和亮点动态加选
- 风险点对应的维度优先级提升

**维度优先级排序逻辑：**
1. 简历风险点对应的维度（最高优先）
2. 岗位类型必选维度
3. 简历亮点可深挖的维度
4. 其他补充维度

### Step 3: 题目生成

对每个选定的维度，生成 2-4 道面试题。

**出题原则：**
- **切中简历**：每道题必须与简历中的具体经历关联，拒绝泛泛而谈
- **有层次**：从浅入深，先事实后分析再判断
- **带追问**：每道题预设 2-3 个追问方向，覆盖不同回答路径
- **可评估**：每道题给出评估标准，面试官可据此判断回答质量

**题目来源：**
- 维度库中的标准问题（可直接使用或改编）
- 针对简历经历定制的个性化问题
- 针对风险点设计的考察题

**追问链设计：**
- 如果候选人回答偏正面 → 追问细节和深度
- 如果候选人回答偏模糊 → 追问具体事例
- 如果候选人回避问题 → 换角度重新切入

### Step 4: 输出整理

按以下结构输出最终结果（格式见 `references/output-template.md`）：

```
## 一、简历分析摘要
（候选人画像 + 关键事件 + 风险点 + 亮点 + 建议考察维度）

## 二、面试题
（按维度分类，每题带考察点、评估标准、追问方向、简历关联）

## 三、追问建议
（针对简历中关键经历的补充追问）
```

## 文件引用

- 评估维度和标准问题 → `references/dimensions.md`
- 简历分析方法 → `references/resume-analysis.md`
- 输出格式模板 → `references/output-template.md`

## 注意事项

1. 面试题要结合简历**具体经历**出题，不能只用维度库中的标准问题
2. 追问链要覆盖候选人可能的不同回答方向
3. 评估标准要具体可操作，面试官能据此判断
4. 风险点对应的维度要重点考察，不能回避
5. 如果简历信息不足，在分析摘要中标注"信息不足"项，对应维度只出通用题
