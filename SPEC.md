# 项目规格说明书 (SPEC)

## 1. 背景与目标

本项目旨在复刻 [ai.maoke123.com](https://ai.maoke123.com/)，构建一个面向电商从业者的 AI 工具聚合平台。平台提供多种 AI 教练/助手工具（覆盖管理、电商运营、内容创作、财税等领域），用户通过对话交互获取专业建议和方案。

项目分两期交付：
- **一期**：AI 对话核心功能 + 工具页面骨架
- **二期**：竞品数据看板 + 用户自定义工具创建

## 2. 用户故事 / 功能需求

### 2.1 一期功能

#### P0 - 核心功能

| 功能 | 描述 |
|------|------|
| AI 通用对话助手 | 用户在输入框提问，AI 实时回复（SSE 流式输出） |
| 多模型切换 | 支持 OpenAI (GPT-4o)、Anthropic (Claude)、Google (Gemini)、国产模型（DeepSeek/通义千问等），用户可在对话前选择模型 |
| Token 用量统计 | 每次对话显示 token 消耗量和估算费用 |
| 文件上传 | 对话中支持上传图片（jpg/png/webp）、文档（PDF/Word/TXT）、表格（Excel/CSV），供 AI 分析 |
| 工具分类首页 | 参考站风格的卡片网格布局，按分类展示 AI 工具入口 |
| 侧边栏 | 展示对话历史、工具分类导航、设置入口 |

#### P1 - 重要功能

| 功能 | 描述 |
|------|------|
| 工具分类骨架 | 预设管理工具、电商工具、设计工具、小红书工具、商业教练、财税工具等分类页面，一期仅"AI 通用助手"可用 |
| 响应式布局 | 桌面优先，基础移动端适配 |
| 流式 Markdown 渲染 | AI 回复支持 Markdown 格式（代码块、表格、列表等），逐字渲染 |

### 2.2 二期功能

| 功能 | 描述 |
|------|------|
| 竞品数据看板 | 淘宝/天猫平台竞品监控，展示 SKU 价格对比、评价分析、销量趋势、商品信息变更 |
| 竞品数据采集 | 混合方式：爬虫抓取 + 第三方 API + 用户手动录入 |
| 数据更新 | 用户手动触发更新（按需模式） |
| ECharts 可视化 | 价格走势图、销量对比图、评价词云等交互式图表 |
| 用户创建工具 | 用户可通过高级配置创建自定义 AI 工具（名称、描述、system prompt、参数、模型选择等） |

### 2.3 暂不做（后续迭代）

- 用户认证系统（账号密码 / 微信 OAuth）
- 付费/积分/会员系统
- 对话历史持久化存储
- 社区互动（评论、点赞、分享）
- 高并发支持

## 3. 技术栈与架构决策

### 3.1 技术栈总览

| 层级 | 技术选型 | 选择理由 |
|------|---------|---------|
| 前端 | React 18 + Vite | 生态成熟，组件库丰富，适合复杂交互；Vite 提供极快的开发体验 |
| 后端 | FastAPI (Python) | Python 生态与 AI/ML 集成方便；FastAPI 原生支持 SSE/async，性能优秀 |
| 数据库 | PostgreSQL | 关系型数据库，数据结构化，支持复杂查询（二期竞品数据需要）；开发期也可用 SQLite 替代 |
| 图表 | ECharts | 功能强大的交互式图表库，适合数据看板 |
| AI 集成 | 直连大模型 API | 后端代理 API Key，前端不暴露密钥 |

### 3.3 AI 模型适配层

后端设计统一的 Model Adapter 接口，各模型提供商实现该接口：

```
ModelAdapter (抽象基类)
├── OpenAIAdapter      → GPT-4o, GPT-4
├── ClaudeAdapter      → Claude Opus/Sonnet/Haiku
├── GeminiAdapter      → Gemini Pro/Flash
└── ChineseModelAdapter → DeepSeek, 通义千问, 文心一言
```

每个 Adapter 负责：
- 请求格式转换
- SSE 流式响应解析
- Token 用量统计

### 3.4 API Key 管理

支持两种模式：
1. **后端代理模式（默认）**：API Key 存储在后端环境变量/配置文件中
2. **用户自带 Key**：用户在设置中填写自己的 API Key，后端加密存储

## 4. 关键 UI/UX 设计点

### 4.1 整体风格

- **设计语言**：简洁现代风 — 大量留白、卡片式布局、渐变色/毛玻璃效果
- **配色**：浅色主题为主，主色调参考蓝色系（#1a73e8），辅以紫色（#6366f1）
- **字体**：系统字体栈，中文优先

### 4.2 页面结构

#### 首页（工具广场）
- 顶部：品牌 Logo + 登录按钮
- 中部：问候语 + 大输入框（支持文件上传、模型选择、发送）
- 下方：分类卡片网格（管理工具、电商工具、设计工具等），每个分类可展开/收起

#### 对话页面
- 左侧：可折叠侧边栏（对话历史 + 智能体列表 + 设置）
- 中间：对话主区域（消息流，支持 Markdown 渲染）
- 底部：输入区（文件上传、模型切换、发送按钮）

#### 竞品看板（二期）
- 顶部：筛选条件（平台、店铺、时间范围）
- 中部：核心指标卡片（价格差异、销量对比、评价变化）
- 下方：ECharts 图表区（趋势图、对比柱状图）

### 4.3 交互要点

- AI 回复采用 SSE 逐字显示，光标闪烁效果
- 侧边栏支持折叠/展开，节省屏幕空间
- 工具卡片 hover 有微动效（阴影/缩放）
- 模型选择下拉菜单，显示模型名称和标识
- 文件上传支持拖拽和点击上传，显示预览缩略图

## 5. 边界情况与错误处理策略

### 5.1 AI 模型调用异常

| 场景 | 处理方式 |
|------|---------|
| API 超时 | 显示"请求超时，请重试"提示，保留用户输入 |
| API 限流 (429) | 显示"请求过于频繁，请稍后重试"，建议切换模型 |
| API Key 无效 | 引导用户检查/更新 API Key |
| 流式中断 | 显示已收到的内容，提示"回复中断"，允许重新生成 |

### 5.2 文件上传异常

| 场景 | 处理方式 |
|------|---------|
| 文件过大 | 限制 20MB，超出时提示 |
| 格式不支持 | 前端校验，即时提示支持的格式 |
| 上传失败 | 显示错误提示，允许重试 |

### 5.3 空状态

- 无对话历史：显示欢迎语和引导提示
- 工具分类为空：显示"暂无工具"占位
- 竞品无数据：显示引导添加竞品的提示

### 5.4 基础错误处理

- 全局错误边界：React Error Boundary 捕获渲染异常
- API 统一错误响应格式：`{ "error": { "code": int, "message": str } }`
- 网络断开检测：navigator.onLine 监听，提示用户检查网络
- 前端 Toast 提示：非阻塞式错误通知

## 6. 非功能性需求

| 维度 | 要求 |
|------|------|
| 性能 | 首屏加载 < 3s（本地环境）；SSE 首字响应 < 5s |
| 安全 | API Key 不暴露给前端；用户自带 Key 加密存储 |
| 可访问性 | 语义化 HTML、键盘导航支持、aria 标签 |
| 浏览器兼容 | Chrome / Edge / Firefox 最新两个大版本 |
| 界面语言 | 纯中文 |
| 响应式 | 桌面优先（1280px+），基础移动端适配 |
| 部署 | 本地部署，单用户使用 |
